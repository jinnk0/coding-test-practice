# [1919] 애너그램 만들기 (Java & Python)

## 🔗 문제 링크
- https://www.acmicpc.net/problem/1919

## 💡 핵심 로직 (Approach)
- 알고리즘: 문자열, 빈도수 배열 (Counting)
- 풀이 전략:
  - 두 문자열이 애너그램이 되기 위해 제거해야 할 문자의 개수가 각 알파벳 출현 빈도의 차이와 같음을 이용함.
  - 크기 26의 배열을 생성하여 첫 번째 문자열의 문자는 더하고(+), 두 번째 문자열의 문자는 빼서(-) 최종적으로 배열에 남은 값들의 절댓값 합을 구함.

## ❌ 삽질 포인트 (Trial & Error)

### 1. 언어별 환경 및 환경 설정 (Java)
- 클래스명 규칙: 백준 등 온라인 저지 환경에서 메인 클래스명을 반드시 Main으로 설정해야 함을 재확인함.
- 입력 방식: Java의 Scanner 객체 생성 및 sc.next() 사용법을 다시 복기함.

### 2. 문법 및 라이브러리 차이
- 라이브러리 명시: 파이썬의 abs()와 달리 자바는 Math.abs()와 같이 Math 클래스 명시가 필요함.
- 데이터 타입 활용: char 타입이 정수형으로 취급되는 자바의 특성상 'a'를 직접 빼는 산술 연산이 가능하여 인덱스 계산이 직관적임.

## ✍️ 언어별 차이 및 복습

| 특징 | Python | Java |
| :--- | :--- | :--- |
| 입력 | input() | Scanner sc = new Scanner(System.in); |
| 문자 -> 숫자 | ord('a') | (int)'a' 또는 'a' - 'a' |
| 절댓값 | abs(-5) | Math.abs(-5) |
| 배열 초기화 | [0] * 26 | new int[26] |

## 🚀 보완할 점 & 더 좋은 해결 방법 (Best Practice)

### 1. sc.next() 사용 권장
- 줄바꿈 문자까지 읽는 nextLine() 대신 단어 단위로 읽는 sc.next()를 사용하여 입력 데이터 뒤의 불필요한 공백 에러를 방지함.

### 2. 성능 최적화: toCharArray()
- 루프 내에서 charAt(i)를 반복 호출하는 대신 toCharArray()를 사용하여 배열로 변환 후 순회하는 것이 성능상 유리함.

```java
for (char c : s1.toCharArray()) {
    alphabet_count[c - 'a']++;
}
```

### 3. 리소스 관리
- Scanner 사용 종료 후 sc.close()를 호출하여 시스템 리소스를 명시적으로 해제하는 습관이 필요함.
